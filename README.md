## Understanding the Dryad Dataset

The dataset has this **actual** structure (not what the README inside it claims):

| Folder/File | What it contains |
|---|---|
| `images_original.zip` | 225 plate images named `1.1.1. original.jpg`, `1.2.1. original.jpg`, etc. |
| `Tables.zip` | 225 `.docx` files named `Table 1.1.1..docx`, `Table 1.2.1..docx`, etc. |
| `Overview GPT JCM.xlsx` | Summary of species + resistance mechanisms per sample |

Each `.docx` table has one row per antibiotic disc with:
- Antibiotic abbreviation (e.g. `CIP`, `MEM`, `AMC`)
- Zone diameter in mm (e.g. `28`)
- Interpretation: `S`, `I`, or `R`
- Breakpoint thresholds

The image and table are linked by **sample ID** (`1.1.1.`).
There are **no x/y disc coordinates** in the data — disc positions are detected automatically via Hough circle transform.

Inside the downloaded Dryad ZIP (`doi_10_5061_dryad_5dv41nsfj__v20241013.zip`)
there are 3 inner ZIPs. Extract them like this:

```
data/dryad/
├── images_original/        ← contents of images_original.zip
│   ├── 1.1.1. original.jpg

│   ├── Table 1.2.1..docx
```

## Step-by-Step Workflow

### Step 1 — Validate your Dryad folders

```bash
python src/inspect_dryad.py \
    --images_dir data/dryad/images_original \
    --tables_dir data/dryad/Tables
```

Expected output:
```
Images found  : 225
Tables found  : 225
Matched pairs : 225
✅  All images have matching tables. Ready to generate masks.
```

---

### Step 2 — Generate masks from Dryad .docx tables

```bash
python src/generate_masks_dryad.py \
    --images_dir data/dryad/images_original \
    --tables_dir data/dryad/Tables \
    --masks_dir  data/dryad/masks \
    --visualise
```

This reads each `.docx` table, extracts zone diameters (mm), detects disc positions
on the plate image using Hough circles, and draws the inhibition zone as a binary mask.

The `--visualise` flag saves `*_vis.jpg` overlays — **open these to verify** the masks
look correct before training. Green = zone, red circles = detected discs.

---

### Step 3 — Auto-generate masks for your ZIP dataset

```bash
python src/generate_masks_zip.py \
    --images_dir data/zip_dataset/images \
    --masks_dir  data/zip_dataset/masks \
    --visualise
```

Fully automatic — no measurements needed. Uses Hough + adaptive thresholding.
Check the `*_vis.jpg` overlays to spot any bad results.

---

### Step 4 — Train the U-Net

#### Recommended (both datasets combined):
```bash
python src/train.py \
    --dryad_images  data/dryad/images_original \
    --dryad_masks   data/dryad/masks \
    --zip_images    data/zip_dataset/images \
    --zip_masks     data/zip_dataset/masks \
    --epochs        60 \
    --batch_size    4 \
    --img_size      512 \
    --model_size    small \
    --output_dir    models/
```

#### Dryad only (if ZIP masks look unreliable):
```bash
python src/train.py \
    --dryad_images  data/dryad/images_original \
    --dryad_masks   data/dryad/masks \
    --epochs        60 \
    --batch_size    4 \
    --model_size    small \
    --output_dir    models/
```

#### Model size guide
| Flag | Params | Recommended for |
|---|---|---|
| `--model_size small` | ~8M | CPU, < 8GB RAM |
| `--model_size full` | ~31M | GPU with 8GB+ VRAM |

---

### Step 5 — Plot Training Curves

```bash
python src/plot_history.py --csv models/training_history.csv
```

---

### Step 6 — Run Inference on a New Plate Image

```bash
python src/predict.py \
    --model_path models/best_model.pth \
    --image      path/to/plate.jpg \
    --output_dir outputs/ \
    --model_size small \
    --guideline  EUCAST
```

Outputs in `outputs/`:
- `<name>_mask.png` — binary segmentation mask
- `<name>_result.jpg` — annotated plate with zone mm + S/I/R labels
- `<name>_measurements.json` — structured results

## Common Issues

| Problem | Fix |
|---|---|
| `No pairs found` | Check both folders exist and names match (e.g. `1.1.1. original.jpg` + `Table 1.1.1..docx`) |
| `ModuleNotFoundError: docx` | Run `pip install python-docx` |
| Hough detects 0 discs | Ensure images are well-lit; try `--visualise` to debug |
| CUDA out of memory | Use `--batch_size 2` or `--img_size 384` |
| Low Dice score | Check masks look correct; reduce `--lr` to `5e-5` |
